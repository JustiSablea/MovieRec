# Следующие задачи по курсовой MovieRec

1. Перенести прототип со статического сайта на backend.
   Нужен Flask или FastAPI: маршруты регистрации, входа, сохранения оценок, получения рекомендаций и деталей фильма.

2. Сделать настоящую базу данных.
   Минимум: `users`, `movies`, `ratings`, `recommendations`, `movie_metadata`. Для курсовой хорошо подойдет SQLite на этапе прототипа и PostgreSQL как более сильный вариант.

3. Реализовать серверную гибридную модель.
   Коллаборативная часть: item-based cosine similarity или SVD. Контентная часть: TF-IDF по жанрам, тегам, описаниям. Итог: `score = alpha * score_cf + (1 - alpha) * score_content`.

4. Подключить обогащение фильмов.
   Через TMDb API получить постеры, описания, актеров, режиссеров и русские описания. Сейчас часть постеров и описаний добавлена только для демонстрации.

5. Сохранить пользовательские оценки.
   После регистрации оценки должны храниться в базе, а не только в `localStorage`. Тогда пользователь сможет вернуться в аккаунт и увидеть прежний профиль.

6. Добавить страницу деталей фильма.
   Нужны постер, описание, жанры, год, средняя оценка MovieLens, причины рекомендации, похожие фильмы и кнопка "Оценить".

7. Провести оффлайн-тестирование модели.
   Разделить MovieLens на train/test и посчитать RMSE, Precision@5, Recall@5, F1@5 для CF, content-based и hybrid, как указано в презентации.

8. Улучшить cold start.
   Для нового пользователя показывать быстрый onboarding с популярными фильмами разных жанров. Для нового фильма использовать только content score.

9. Доработать браузерное расширение.
   Оно должно распознавать название фильма на открытой странице, показывать похожие фильмы, открывать карточку MovieRec и отправлять оценку в аккаунт.

10. Подключить нейросетевой поиск по описанию.
    Пользователь вводит описание вроде "фильм про сны и ограбление", модель возвращает вероятные фильмы. Это можно сделать через embeddings или отдельный API-эндпоинт.

11. Подготовить демонстрационный сценарий защиты.
    Сценарий: регистрация, добавление 3 фильмов, объяснение CF/content score, получение рекомендаций, просмотр деталей, сравнение с метриками из эксперимента.

12. Оформить проектную документацию.
    Нужны схема архитектуры, схема БД, описание алгоритма, инструкция запуска, описание ограничений и план дальнейшего развития.
